## Overview

The KBB application uses Yahoo OAuth 2.0 for user authentication. The callback URL is **automatically generated at runtime** based on your environment, eliminating the need for manual configuration of a separate `YAHOO_CALLBACK_URL` environment variable.

### Key Points

- ✅ Callback URL is **automatically generated** from `NEXTAUTH_URL` or `VERCEL_URL`
- ✅ No need to manually set `YAHOO_CALLBACK_URL` in environment variables
- ✅ Works seamlessly across local development, staging, and production environments
- ✅ Debug logging available to verify the callback URL being used

## Dynamic Callback URL System

### How It Works

The callback URL is generated by the `getYahooCallbackUrl()` function in `lib/get-callback-url.ts`. It follows this resolution order:

1. **NEXTAUTH_URL** (if set) — Uses the explicitly configured base URL
2. **VERCEL_URL** (if set) — Uses Vercel's automatically provided URL for preview deployments
3. **Fallback** — Defaults to `http://localhost:3000` for local development

The callback path `/api/auth/callback/yahoo` is always appended to the resolved base URL.

### Resolution Examples

| Environment | NEXTAUTH_URL | VERCEL_URL | Result |
|---|---|---|---|
| Local Dev | `http://localhost:3000` | (not set) | `http://localhost:3000/api/auth/callback/yahoo` |
| Local Dev | (not set) | (not set) | `http://localhost:3000/api/auth/callback/yahoo` (fallback) |
| Production | `https://yourdomain.com` | (not set) | `https://yourdomain.com/api/auth/callback/yahoo` |
| Vercel Preview | (not set) | `my-app-abc123.vercel.app` | `https://my-app-abc123.vercel.app/api/auth/callback/yahoo` |
| Vercel Production | `https://yourdomain.com` | `yourdomain.vercel.app` | `https://yourdomain.com/api/auth/callback/yahoo` (NEXTAUTH_URL takes precedence) |

## Local Development Setup

### Step 1: Create Yahoo OAuth Application

1. Go to [Yahoo Developer Console](https://developer.yahoo.com/apps)
2. Sign in with your Yahoo account (create one if needed)
3. Click "Create an app"
4. Fill in the application details:
   - **Application Name**: KBB (or your preferred name)
   - **Application Type**: Web Application
5. Click "Create app"

### Step 2: Configure Redirect URIs

1. In your application settings, find the "Redirect URIs" section
2. Add the following redirect URI for local development:
   ```
   http://localhost:3000/api/auth/callback/yahoo
   ```
3. Save the changes

### Step 3: Get OAuth Credentials

1. In your application settings, find the "OAuth 2.0 Credentials" section
2. Copy the following values:
   - **Client ID** (also called "Consumer Key")
   - **Client Secret** (also called "Consumer Secret")

### Step 4: Configure Environment Variables

Create or update your `.env.local` file in the project root:

```bash
# Generate a secure secret
openssl rand -base64 32

# Add to .env.local
NEXTAUTH_SECRET=<generated_secret_from_above>
NEXTAUTH_URL=http://localhost:3000

# Yahoo OAuth credentials (from Step 3)
YAHOO_AUTH_URL=https://api.login.yahoo.com/oauth2/request_auth
YAHOO_TOKEN_URL=https://api.login.yahoo.com/oauth2/get_token
YAHOO_CLIENT_ID=<your_client_id>
YAHOO_CLIENT_SECRET=<your_client_secret>
```

### Step 5: Verify Setup

1. Start the development server:
   ```bash
   yarn dev
   ```

2. Check the console output for debug logs:
   ```
   [getYahooCallbackUrl] Using NEXTAUTH_URL: http://localhost:3000
   [getYahooCallbackUrl] Generated callback URL: http://localhost:3000/api/auth/callback/yahoo
   [NextAuth] Yahoo OAuth callback URL: http://localhost:3000/api/auth/callback/yahoo
   ```

3. Navigate to the login page and test the Yahoo OAuth flow

## Production Setup

### Step 1: Update Yahoo OAuth Application Settings

1. Go to [Yahoo Developer Console](https://developer.yahoo.com/apps)
2. Select your application
3. In the "Redirect URIs" section, add your production domain:
   ```
   https://yourdomain.com/api/auth/callback/yahoo
   ```
4. Save the changes

### Step 2: Configure Environment Variables

Set the following environment variables in your production environment (e.g., GitHub Secrets, environment configuration service):

```bash
# Production domain
NEXTAUTH_URL=https://yourdomain.com

# Generate a secure secret for production
openssl rand -base64 32

# Add to production environment
NEXTAUTH_SECRET=<generated_secret>

# Yahoo OAuth credentials (same as local development)
YAHOO_AUTH_URL=https://api.login.yahoo.com/oauth2/request_auth
YAHOO_TOKEN_URL=https://api.login.yahoo.com/oauth2/get_token
YAHOO_CLIENT_ID=<your_client_id>
YAHOO_CLIENT_SECRET=<your_client_secret>
```

### Step 3: Verify Deployment

After deploying to production:

1. Check the application logs for debug output
2. Test the Yahoo OAuth login flow
3. Verify that the callback URL matches what's registered in Yahoo's Developer Console

## Vercel Deployment Setup

### Step 1: Update Yahoo OAuth Application Settings

1. Go to [Yahoo Developer Console](https://developer.yahoo.com/apps)
2. Select your application
3. In the "Redirect URIs" section, add your Vercel deployment URLs:
   - For preview deployments: `https://*.vercel.app/api/auth/callback/yahoo` (if supported)
   - Or add specific preview URLs as needed: `https://my-app-preview-abc123.vercel.app/api/auth/callback/yahoo`
   - For production: `https://yourdomain.com/api/auth/callback/yahoo`

### Step 2: Configure Vercel Environment Variables

1. Go to your Vercel project settings
2. Navigate to "Environment Variables"
3. Add the following variables:

**For Production Deployments:**
```
NEXTAUTH_URL=https://yourdomain.com
NEXTAUTH_SECRET=<generated_secret>
YAHOO_AUTH_URL=https://api.login.yahoo.com/oauth2/request_auth
YAHOO_TOKEN_URL=https://api.login.yahoo.com/oauth2/get_token
YAHOO_CLIENT_ID=<your_client_id>
YAHOO_CLIENT_SECRET=<your_client_secret>
```

**For Preview Deployments (Optional):**
- Leave `NEXTAUTH_URL` unset
- The system will automatically use `VERCEL_URL` provided by Vercel
- Other variables remain the same

### Step 3: How Vercel URLs Work

Vercel automatically provides the `VERCEL_URL` environment variable for each deployment:

- **Production**: `yourdomain.vercel.app` (or your custom domain if configured)
- **Preview**: `my-app-pr-123-abc123.vercel.app` (unique URL for each PR)

The callback URL is automatically generated as:
```
https://<VERCEL_URL>/api/auth/callback/yahoo
```

### Step 4: Verify Deployment

After deploying to Vercel:

1. Check the Vercel deployment logs for debug output
2. Test the Yahoo OAuth login flow on your preview/production URL
3. Verify that the callback URL matches what's registered in Yahoo's Developer Console

## Troubleshooting

### Issue: "Invalid redirect_uri" Error from Yahoo

**Cause**: The callback URL being sent to Yahoo doesn't match what's registered in Yahoo's Developer Console.

**Solution**:
1. Check the debug logs in your application console:
   ```
   [getYahooCallbackUrl] Generated callback URL: <actual_url>
   ```
2. Verify this URL is registered in Yahoo's Developer Console
3. Ensure there are no trailing slashes or protocol mismatches
4. Check that `NEXTAUTH_URL` is set correctly for your environment

### Issue: Callback URL is `http://localhost:3000` in Production

**Cause**: `NEXTAUTH_URL` is not set in your production environment.

**Solution**:
1. Set `NEXTAUTH_URL` to your production domain in your environment variables
2. For Vercel, either set `NEXTAUTH_URL` or ensure `VERCEL_URL` is available
3. Restart your application after updating environment variables

### Issue: Debug Logs Not Appearing

**Cause**: Console output might be filtered or not visible in your environment.

**Solution**:
1. For local development, check the terminal where you ran `yarn dev`
2. For production, check your application logs (e.g., Vercel logs, server logs)
3. Ensure the application is properly initialized (logs appear during startup)

### Issue: OAuth Flow Fails After Redirect

**Cause**: Multiple possible causes including mismatched callback URLs, expired credentials, or network issues.

**Solution**:
1. Verify the callback URL matches Yahoo's registered URL
2. Check that `YAHOO_CLIENT_ID` and `YAHOO_CLIENT_SECRET` are correct
3. Ensure `NEXTAUTH_SECRET` is set and consistent
4. Check browser console for error messages
5. Review application logs for detailed error information

## FAQ

### Q: Do I need to set YAHOO_CALLBACK_URL in my environment?

**A**: No. The callback URL is automatically generated at runtime by `lib/get-callback-url.ts`. You should NOT set `YAHOO_CALLBACK_URL` manually.

### Q: What if I need different callback URLs for different environments?

**A**: The system handles this automatically:
- Set `NEXTAUTH_URL` to your domain for each environment
- Or let Vercel's `VERCEL_URL` handle it for preview deployments
- The callback URL is generated dynamically based on these values

### Q: Can I use a custom domain with Vercel?

**A**: Yes. Set `NEXTAUTH_URL` to your custom domain in your Vercel environment variables. This takes precedence over `VERCEL_URL`.

### Q: How do I test the callback URL locally?

**A**: Check the debug logs when starting your development server:
```bash
yarn dev
```

Look for output like:
```
[getYahooCallbackUrl] Using NEXTAUTH_URL: http://localhost:3000
[getYahooCallbackUrl] Generated callback URL: http://localhost:3000/api/auth/callback/yahoo
[NextAuth] Yahoo OAuth callback URL: http://localhost:3000/api/auth/callback/yahoo
```

### Q: What if NEXTAUTH_URL is not set and I'm not on Vercel?

**A**: The system falls back to `http://localhost:3000`. This is suitable for local development but should not be used in production.

### Q: How often is the callback URL generated?

**A**: The callback URL is generated every time the NextAuth configuration is loaded (typically once per application startup). It's not cached, so environment changes take effect after restarting the application.

### Q: Can I use HTTP in production?

**A**: It's not recommended. Always use HTTPS (`https://`) in production for security. HTTP should only be used for local development.

### Q: What if I have multiple applications?

**A**: Create separate OAuth applications in Yahoo's Developer Console for each application. Each application should have its own `YAHOO_CLIENT_ID` and `YAHOO_CLIENT_SECRET`.
